## PARTE 1 — A FÁBRICA: O QUE É A IAH

### O Problema que Resolvemos

```
AGÊNCIA TRADICIONAL
  ├── 15 pessoas (developers, designers, PM, QA, comercial)
  ├── Custo fixo: R$80.000–300.000/mês
  ├── Prazo: 6–18 meses
  ├── Taxa de falha: 70% dos projetos nunca vão ao ar
  └── Preço mínimo para o cliente: R$50.000

FREELANCER
  ├── 1 pessoa, conhecimento limitado
  ├── Sem visão de negócio — executa o que pedem
  ├── Sem metodologia — cada projeto começa do zero
  └── Entrega código. Não entrega produto.

IAH — INTELIGÊNCIA ARTIFICIAL HUMANA (Pentalateral)
  ├── 1 Diretor + 4 IAs especializadas (5 atores no total)
  ├── Custo fixo: R$350/mês (infraestrutura completa)
  ├── Prazo: V1 em 48h, produto completo em 1–4 semanas
  ├── Taxa de sucesso: 100% (o projeto não para)
  ├── 25 ideias/ciclo [M×2+G+N+E × 5]: [M+E+G+N × 5] — inteligência composta acumulada
  └── Preço: calibrado ao valor gerado, não ao custo de produção
```

### A Linha de Montagem

```
                        ┌──────────────────────────────────────┐
   QUALQUER PROJETO →   │     CONSELHO PENTALATERAL IAH         │   → PRODUTO REAL
                        │                                      │
   E-commerce?          │  DIRETOR          GEMINI             │   Funcionando.
   App mobile?          │  (Veredito)       (Estrategista)     │   No ar.
   SaaS?                │                                      │   Documentado.
```

**O que diferencia a fábrica da agência:** A fábrica não contrata mais gente para produzir mais. Ela otimiza o processo. Cada versão torna o próximo projeto mais rápido. O conhecimento fica no git — nunca sai quando um funcionário pede demissão.

## PARTE 2 — O CONSELHO: PAPÉIS IMUTÁVEIS (PENTALATERAL)

> **Atualizado em 2026-05-18 — Sistema evoluiu de Quadrilateral para Pentalateral IAH**
> O Embaixador é o 4º membro ativo — incorporado formalmente em 2026-05-17.

| Membro | Papel | O que faz | O que NÃO faz |
|---|---|---|---|
| **Diretor** | O Veredito | Define ROI, contexto de negócio, aprova entregas, fala com o cliente | Não escreve código. Não desenha UI. |
| **Gemini** | O Estrategista | Analisa mercado, cria DIRETRIZ, pesquisa referências, gera conteúdo | Não executa código. Não audita contexto passado. |
| **NotebookLM** | O Auditor | Guarda o histórico completo do projeto, audita decisões, gera a Skill técnica | Não decide. Não cria estratégia. |
| **Claude Code** | O Músculo | Lê a Skill, escreve o código, cria banco de dados, entrega o software | Não define estratégia de mercado. |
| **Embaixador** | Inteligência Persistente do Cliente | Acumula memória do cliente entre sessões, filtra realidade das ideias do Conselho, monitora saúde do relacionamento | Não executa código. Não substitui o Veredito do Diretor. |

**Regra de ouro:** Nenhuma IA substitui a outra. A força está na restrição dos papéis.

**Singularidade do Embaixador:** É o único membro com **memória persistente por cliente** — acumula inteligência a cada ativação (sessão 50 sabe 50x mais que sessão 1). Aplica o **Filtro de Realidade (P-031)**: CONFIRMA, EXPANDE ou ALERTA cada ideia gerada pelos outros membros com base no comportamento real observado do cliente. Opera em Claude Projects com memória nativa.

**25 ideias/ciclo [M×2+G+N+E × 5]:** Músculo [M-1..M-5] + Embaixador [E-1..E-5] + Estrategista [G-1..G-5] + Auditor [N-1..N-5]. Cada ciclo completo gera 20 ideias que alimentam o próximo loop — inteligência composta que se acumula exponencialmente.

## PARTE 3 — A MATRIZ DE 5 CAMADAS: QUALQUER PROJETO, SEMPRE A MESMA ESTRUTURA

A descoberta central: **todo produto digital evolui pelas mesmas 5 camadas**, independente do nicho. O que muda é o conteúdo, não a estrutura.

```
┌─────────────────────────────────────────────────────────────────┐
│  CAMADA 5 — MONOPÓLIO        V17–V18                            │
│  Dados proprietários · Efeitos de rede · Canal institucional    │
├─────────────────────────────────────────────────────────────────┤
│  CAMADA 4 — ESCALA           V12–V16                            │
│  Multi-tenant · White-label · Dashboard avançado · Segurança    │
├─────────────────────────────────────────────────────────────────┤
│  CAMADA 3 — INTELIGÊNCIA     V8–V11                             │
│  IA no produto · Automação · Análise de dados · Webhooks        │
├─────────────────────────────────────────────────────────────────┤
│  CAMADA 2 — MOTOR            V4–V7                              │
│  Auth · Funcionalidade core · Pagamento · API externa           │
├─────────────────────────────────────────────────────────────────┤
│  CAMADA 1 — FUNDAÇÃO         V1–V3                              │
│  Interface pública · Captura de dados · Banco · Deploy          │
└─────────────────────────────────────────────────────────────────┘
```

### Como Aplicar a Matriz em Qualquer Projeto

**Exemplo 1 — Clínica Médica quer app de agendamento**
```
Camada 1: PWA de agendamento (calendário + formulário) → Deploy → Cliente vê em 48h
Camada 2: Login do paciente + painel da clínica + Stripe para consultas pagas
Camada 3: Lembrete automático via WhatsApp + análise de horários mais procurados
Camada 4: Múltiplas unidades da clínica + white-label para outras clínicas
Camada 5: Dados de saúde agregados → produto B2B para planos de saúde
```

**Exemplo 2 — Artesão quer loja online**
```
Camada 1: E-commerce PWA com produtos + formulário de contato → 3 dias
Camada 2: Carrinho + Pix/Stripe + painel do vendedor
Camada 3: Recomendação de produtos por IA + chat de suporte automático
Camada 4: Marketplace (outros artesãos vendem na mesma plataforma)
Camada 5: Dados de tendências de artesanato → produto B2B para atacadistas
```

**Exemplo 3 — Consultoria quer sistema de orçamentos**
```
Camada 1: Formulário de briefing + gerador de PDF → 2 dias
Camada 2: Login + histórico de orçamentos + aprovação digital do cliente
Camada 3: IA sugere itens do orçamento + calcula margem automaticamente
Camada 4: Múltiplos consultores na mesma plataforma + relatórios gerenciais
Camada 5: Benchmark de preços do setor → produto para associações da categoria
```

**O cliente pode entrar em qualquer camada e sair em qualquer camada.** O preço varia conforme.

## PARTE 4 — O PROCESSO: DO PEDIDO À ENTREGA

### Fase 0 — DIRETRIZ ZERO (Discovery)
**Duração:** 30–60 min com o cliente  
**Quem:** Diretor + cliente

O Diretor conduz 7 perguntas:
```
1. PROBLEMA REAL   → "Qual dor do negócio este produto vai resolver?"
2. QUEM USA        → "Quem usa no dia a dia? Descreva essa pessoa."
3. SUCESSO         → "Como sabe que deu certo? O que muda no negócio?"
4. HOJE            → "O que usa hoje para isso? O que não funciona?"
5. REFERÊNCIAS     → "Mostra 2 produtos que admira (qualquer setor)."
6. RESTRIÇÕES      → "O que não pode existir? Prazo fixo? Integração obrigatória?"
7. ORÇAMENTO       → "Qual envelope disponível? (para calibrar a camada de entrada)"
```

**Output:** Briefing de 1 página → alimenta o NotebookLM como Notebook do cliente.

---

### Fase 1 — DIRETRIZ V1 (Gemini → NotebookLM → Claude)

```
GEMINI recebe o briefing do Discovery:
  → Analisa 3 concorrentes diretos
  → Propõe posicionamento visual (paleta, tipografia, referências)
  → Escreve a DIRETRIZ V1 (o que construir, como diferenciador)

NOTEBOOKLM indexa:
  → Documentos do cliente (manual, regulamento, catálogo)
  → Output do Gemini
  → Gera a Skill técnica para o Claude

CLAUDE CODE lê a Skill:
  → Registra AUTO-LOG
  → Constrói Camada 1 (fundação)
  → Commit + deploy → cliente vê algo real em 48h
```

---

### Fase 2 — ITERAÇÃO RÁPIDA (Sprints de 48h)

Cada Sprint segue o mesmo ciclo:
```
[Diretor] Define a próxima feature → [Claude] Implementa → [Diretor] Valida com cliente
```

**Cadência esperada:**
- Dia 1–2: V1 online (Camada 1)
- Dia 3–5: V2–V3 (funcionalidade core)
- Semana 2: V4–V6 (Camada 2 completa)
- Semana 3–4: V7–V10 (Camada 3, se contratado)

---

### Fase 3 — ENTREGA E HANDOFF

```
[ ] Deploy em produção com domínio do cliente
[ ] Documentação de uso (gerada pelo Claude — 1h)
[ ] MEMORIA_VX.md do projeto (registo técnico permanente)
[ ] Videochamada de treinamento (1h) — o cliente aprende a usar
[ ] Credenciais e acessos entregues (git, Supabase, painel)
[ ] Proposta de retainer: "quer continuar evoluindo o produto?"
```

---

### Fase 4 — RETAINER EVOLUTIVO (opcional, mas ideal)

O cliente paga mensalmente para continuar evoluindo o produto. Para a fábrica, é receita previsível. Para o cliente, é produto que nunca para de melhorar.

Cada mês = 1 Sprint de evolução = 1 nova camada ou funcionalidade.

## PARTE 5 — CLONE DE PROJETO: COMO USAR O VANGUARD COMO MOLDE

A plataforma Vanguard (V1–V16) não é apenas o produto interno. É o **molde** para qualquer projeto de cliente. O processo de clonar:

```
1. O cliente define o projeto (Discovery)
2. O Diretor identifica em qual camada entrar (1, 2 ou 3)
3. Claude clona o boilerplate da camada equivalente:
   - Precisa de auth + payments? → base da V6
   - Precisa de scanner/análise? → base da V12
   - Precisa de multi-tenant?   → base da V8
4. Gemini adapta o design visual para a marca do cliente
5. Claude adapta as variáveis (nome, cores, funcionalidades específicas)
6. Deploy em 48h com identidade do cliente
```

**Resultado:** O que levaria 3 meses do zero leva 1 semana clonando a base Vanguard.

## PARTE 7 — PRECIFICAÇÃO: COMO COBRAR PELO VALOR, NÃO PELO TEMPO

### Princípio

> *Nunca diga "custa tanto por hora". Diga "este produto vai gerar tanto para o seu negócio. O investimento é tanto."*

**Âncoras de valor:**
- Um e-commerce que vende R$30.000/mês → o site vale R$5.000–15.000
- Um app que economiza 40h/mês de trabalho manual → vale R$3.000–8.000/mês de economia → R$5.000 de uma vez é desconto
- Uma plataforma multi-tenant com 20 clientes a R$200/mês → gera R$4.000/mês → vale R$20.000

### Tabela de Produtos

| Produto | Camadas | Prazo | Preço |
|---|---|---|---|
| **Sprint Discovery** | Análise + wireframe + V1 online | 3–5 dias | R$1.500–3.000 |
| **Produto Lançado** | Camadas 1+2 (V1–V6) | 2–4 semanas | R$5.000–12.000 |
| **Plataforma Inteligente** | Camadas 1+2+3 (V1–V11) | 4–8 semanas | R$12.000–25.000 |
| **Plataforma Enterprise** | Camadas 1–4 (V1–V16) | 2–4 meses | R$25.000–60.000 |
| **Retainer de Evolução** | 1 Sprint/mês de novas features | Mensal | R$2.500–6.000/mês |
| **Equity Deal** | Build completo por participação | Acordo | 5–20% do negócio |

### Regras de Precificação

1. **Nunca cobre menos que R$1.500** — abaixo disso o cliente não valoriza
2. **Sempre proponha o retainer** — é onde está a receita recorrente
3. **Cobrar mais por urgência** — prazo de 48h tem 30% de adicional
4. **Equity apenas em projetos com potencial de R$1M+** — senão é custo disfarçado de crença

## PARTE 9 — COMUNICAÇÃO: COMO VENDER SEM VENDER

### O que o mercado entende

❌ "Uso IA para desenvolver software mais barato"  
✅ "Entregamos produtos digitais completos em semanas, não meses"

❌ "Tenho um método com 4 IAs"  
✅ "Temos uma metodologia proprietária de engenharia de produto"

❌ "É mais barato que uma agência"  
✅ "Você investe R$X e tem um produto que gera R$Y por mês"

### Os 3 canais que funcionam (na ordem)

**1. Portfolio vivo (antes de qualquer anúncio)**  
A plataforma Vanguard é a demonstração. "Construímos isso para o nosso próprio negócio em 16 semanas." O cliente entende capacidade sem que você precise explicar.

**2. Indicação ativa (o motor de crescimento)**  
Todo cliente entregue recebe: "Você conhece alguém com uma necessidade similar? Cada indicação que fechar tem 10% de comissão para você."

**3. LinkedIn do Diretor (construção de autoridade)**  
1 post por semana com antes/depois de um projeto. Screenshot do produto funcionando. Não texto — produto visual. O algoritmo distribui, os leads chegam.
