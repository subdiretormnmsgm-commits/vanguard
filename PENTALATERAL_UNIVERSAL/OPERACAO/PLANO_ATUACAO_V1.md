## REGRAS INEGOCIÁVEIS

1. **NF-e nunca custom** — sempre via API off-the-shelf (NFE.io, Tiny, Omie)
2. **Anúncio pago só após case study real publicado**
3. **Protocolo de Emergência 24h só após o segundo cliente**
4. **Escopo travado por escrito antes de qualquer build começar**
5. **ROI_TRACKER só com números reais — jamais estimativas artificiais**

## PARCEIROS ESTRATÉGICOS

**Alvo:** Contadores · Coaches · Consultores de RH
**Não alvo:** Agências (concorrência direta)

**Por quê funciona:** parceiro tem carteira de clientes com dor digital. Nós resolvemos. Ele complementa, não compete.

**Modelo:** indicação → comissão de 10–15% sobre projeto fechado.

## SEQUÊNCIA DE BUILD APROVADA

```
1. V1 — Case Study (amigo do Eduardo)
   → Valida o processo completo de ponta a ponta
   → Testa o Playbook automatizado pela primeira vez
   → Gera os primeiros números reais

2. Kit Contador
   → Dashboard financeiro (Supabase + Chart.js)
   → Contas a pagar/receber
   → NF-e via API off-the-shelf
   → FIRE Event: "pagamento registrado"

3. Diagnóstico Financeiro Rápido (Isca do Contador)
   → Ferramenta que o Contador usa com o cliente dele
   → 5 perguntas → hemorragia estimada → CTA WhatsApp Eduardo
   → Build: 1 dia

4. White Label Playbook
   → brand-config.json por parceiro/cliente
   → Template renderiza com marca do parceiro
   → Build: 2–3h no template existente
```
