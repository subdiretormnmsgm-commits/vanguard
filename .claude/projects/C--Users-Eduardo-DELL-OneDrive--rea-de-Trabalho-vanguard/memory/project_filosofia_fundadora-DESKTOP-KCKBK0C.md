O negócio Quadrilateral IAH opera sobre um axioma declarado pelo Diretor:

> "A gestão de tudo no nosso negócio deve ser realizada com gestão de processos, evolução e inteligência — este é o nosso maior ativo."

**Why:** Não é um SaaS de produto. É uma operação onde o protocolo, o processo e a inteligência acumulada (memória das IAs, templates, frameworks) são a vantagem competitiva real. O produto entregue ao cliente é consequência — o ativo é o sistema que o produz.

**How to apply:** Toda decisão de build, toda sessão do Conselho, todo fechamento de versão deve ser avaliada pela pergunta: "isso torna o nosso sistema mais inteligente e mais evoluído?" Se não, não é prioridade.
