# Skill: vanguard-v14-optimization
**Descrição:** Evolução da Vanguard Tech para a V14 (Sovereign Optimization & Feedback Loop).

## 1. 🎯 Objetivo
Transformar o ecossistema Vanguard numa máquina adaptativa orientada pelo ciclo PDCA (Ajuste). A missão desta operação é criar o Livro de Razão de Performance, implementar a Inteligência de Colmeia (Hive Mind) no Hermes, desenvolver geradores visuais de alto impacto (Trojan Outbound), tornar o HUD adaptativo e executar uma rigorosa consolidação da nossa base de conhecimento.

## 2. 🧠 Leis do Conselho & Engenharia de Otimização
- **Performance Ledger:** Crie e preencha o arquivo `VANGUARD_PERFORMANCE_LEDGER.md`. Registe a hipótese de cada versão anterior (foco na V11 a V13), resultados técnicos, métricas de conversão e dívidas técnicas geradas. Ele servirá como a nossa bússola analítica.
- **Hermes Hive Mind:** Implemente a lógica de inteligência coletiva no banco de dados e na API. Os templates do Hermes com maior taxa de sucesso devem ser automaticamente promovidos a "Vanguard Recommended" para os tenants do mesmo nicho, garantindo que a rede aprenda como uma colmeia.
- **Trojan Outbound Previews:** Desenvolva o gerador (V1) de assets visuais de alto impacto (vídeo/GIFs do HUD personalizado). Este motor extrairá o impacto do scanner visual para envio via WhatsApp Business API, superando a barreira do texto.
- **Dynamic HUD Adjustment:** Refatore o layout "Bento Grid Cockpit" (criado na V12). O Dashboard deve ser adaptativo e contextual, alterando os módulos e alertas exibidos consoante o "Maturity Score" do lead detetado no momento.
- **Consolidação de Memória & Autocura:** 
    - Realize uma auditoria nos arquivos `.md` históricos e proponha no Knowledge Graph um plano de 'Consolidação de Memória' para manter o contexto do Claude ágil.
    - Atualize a Constituição (`VANGUARD_BUSINESS_RULES.md`) para formalizar as regras do SLA de Autocura (notificação automática de falhas < 60s).
    - Adicione a seção 'Fluxo de Dados de Retroalimentação' no `VANGUARD_KNOWLEDGE_GRAPH.md`.

## 3. ⚙️ Execução & Gestão de Conhecimento
O workflow obrigatório é: `Brainstorm` -> `Plan` -> `Code (SQL Hive Mind, Trojan Generator, Adaptive HUD, Markdown ledgers)` -> `Code Review` -> `Git Commit`.
**Registo de Memória Obrigatório:**
1. Crie o arquivo `memorias/MEMORIA_14_OPTIMIZATION.md` detalhando as arquiteturas adaptativas geradas.
