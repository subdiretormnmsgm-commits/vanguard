[G-1 a G-5]: 5 IDEIAS DISRUPTIVAS DO ESTRATEGISTA PARA LOOP 7

* **[G-1] Algoritmo Safe-Horizon (SM-2 Adaptativo para Mudança de Edital)**: Se a data do concurso SEDES-DF for alterada para 2027, o sistema deve acionar automaticamente uma trava de desaceleração que expande os intervalos do SM-2 e reduz a carga diária do feed de 20 para 5 questões de revisão essencial, mudando o foco do produto de "tiro curto de urgência" para "manutenção de longo prazo", evitando o esgotamento da usuária.


* **[G-2] Dry-Run de Isolamento de Tenant (`test_tenant_isolation.ps1`)**: Um script automatizado que simula a ingestão concorrente de dados de um usuário malicioso tentando injetar registros com `user_id` alterado, validando fisicamente na camada do banco se o RLS bloqueia o vazamento de dados antes de darmos o GO para o segundo cliente do mesmo nicho.


* **[G-3] Mapeador de Replicabilidade Quadrix (Próximo Nicho Alvo)**: Uma função analítica interna que cruza a estrutura do arquivo `edital_sedes.json` com os editais abertos do Distrito Federal organizados pela mesma banca (Quadrix), apontando estatisticamente qual o próximo cargo público com 80%+ de similaridade programática para direcionar o pipeline comercial ativo da Vanguard.


* **[G-4] Telemetria Passiva por Evento de Componente (Heartbeat Silencioso)**: Inserir no frontend do PWA um listener assíncrono leve que atualiza a tabela `evento_uso` a cada mudança de estado dos componentes F-7 (Raio-X SVG) e F-8 (Termômetro), registrando o comportamento real da cliente sem utilizar trackers de terceiros, preservando a soberania absoluta dos dados.


* **[G-5] Git Hook pre-push Antiamnésia de Processo (Enforcement do P-045)**: Criar um script de automação de repositório que intercepta a ação de push do desenvolvedor; se o script detectar que o loop atual mudou no `WIP_BOARD.json` mas os arquivos `MEMORIA_V*.md` e `relatorio_evolutivo_V*.md` correspondentes não existem na pasta histórica, o commit/push é abortado fisicamente, punindo no processo a negligência documental.
