### BLOCO 0 — DIAGNÓSTICO

O risco crítico invisível que Músculo e Diretor estão ignorando nesta transição de infraestrutura é o **Apagão Administrativo de Billing**. Transferir a API Key do Gemini para a conta do Valdece sem um espelhamento de contingência cria uma dependência cega: se o cartão corporativo do cliente falhar na validação do Google Cloud, o sistema cai em produção instantaneamente.

Além disso, a migração para a região `sa-east-1` (São Paulo) pode alterar sutilmente o comportamento de roteamento da API. Executar a migração sem rodar o freeze técnico prévio no banco Vanguard viola o princípio `[P-038]` e pode mascarar incompatibilidades latentes no script `v3_migration.sql`.

### BLOCO 1 — PRIORIDADES DE BUILD (MÁXIMO 3)

1. **Isolamento da API Key via Supabase Edge Function (HV-1 Fix)**
* *O que construir:* Mover a lógica de geração de embeddings do frontend vanilla para uma Edge Function dedicada no Supabase, utilizando o token armazenado com segurança no Vault do servidor.

* *O que fica fora:* Logs avançados de consumo e painel de controle de tokens por usuário.


2. **Renderização dos Badges Vinculantes e Atualização da Citação ABNT**

* *Por que agora:* É o cerne da entrega da V3 contratada. O valor percebido pelo advogado penalista escala drasticamente ao bater o olho no peso do precedente durante a pressão de uma audiência.
* *Horas reais:* 2,5 horas (ajuste de CSS no tema Navy/Ouro + manipulação de string na função de cópia ABNT).
* *O que fica fora:* Seleção múltipla de cards ou exportação de blocos de acórdãos de uma só vez.


3. **Migração Soberana de Infraestrutura para sa-east-1 (P-013)**

* *Por que agora:* Contrato assinado exige a entrega da soberania total do ecossistema e o encerramento do uso de servidores internos da Vanguard.
* *Horas reais:* 4 horas (configuração do tenant + dump/restore + execução completa do checklist P-038 com as 12 queries de validação técnica).

### BLOCO 6 — 5 IDEIAS DISRUPTIVAS DO ESTRATEGISTA

* **`[G-1]` `[CONTRA-INTUITIVO]` Cache Estratégico de Embeddings na Edge Function:** Salvar no Supabase os embeddings das 20 queries mais realizadas pelo Valdece. Quando ele repetir uma busca em plenário, o sistema intercepta e entrega o resultado em < 100ms, sem gastar processamento na API do Gemini e gerando um efeito de velocidade absurda diante do juiz.
* *Pergunta ao Músculo:* Qual o impacto estrutural de criar uma tabela `query_cache` para interceptar a requisição antes do disparo da Edge Function?


* **`[G-2]` `[CONTRA-INTUITIVO]` Demonstração Oculta de Resiliência (Fail-Open Guard):** Injetar um script de simulação que desconecte a API por 5 segundos durante os testes de homologação. O sistema deve falhar de forma elegante, exibindo os últimos resultados salvos em cache local no dispositivo do cliente, provando que o Toga Digital não deixa o advogado estagnado se a internet do tribunal cair.
* *Pergunta ao Músculo:* O service worker atual do PWA consegue sustentar a exibição dos cards de resultado offline baseando-se no LocalStorage?


* **`[G-3]` Citação ABNT com Link de Validação Soberana:** Modificar a string copiada pelo botão "Copiar ABNT" para incluir uma URL curta e pública apontando para a ementa completa hospedada no Supabase do próprio cliente (ex: `valdece.toga.digital/v/hc-512290`). O juiz ou assessor poderá clicar na petição e abrir o acórdão validado diretamente na infraestrutura do escritório.
* *Pergunta ao Músculo:* Conseguimos criar uma rota dinâmica simples no frontend vanilla para renderizar o card isolado do acórdão a partir do ID da URL?


* **`[G-4]` Indexador Automático de Peso por Turma Julgadora:** Utilizar o campo `turma` adicionado no schema V3 para injetar um multiplicador de relevância silencioso. Se o Valdece estiver buscando uma tese e houver um acórdão da Turma específica perante a qual ele vai sustentar oralmente, esse card ganha um acréscimo de `+0.10` no score de similaridade.
* *Pergunta ao Músculo:* É possível passar o parâmetro da Turma alvo na chamada da RPC para modular o cálculo do cosseno em tempo real?


* **`[G-5]` Painel Minimalista de Telemetria de ROI para o Diretor:** Criar uma view SQL protegida (`view_diretor_roi`) que calcule automaticamente: (número de buscas realizadas pelo cliente × R$ 200 de valor/hora economizada). ### INTERAÇÃO LIVRE

1. **O Risco Oculto do Slip de Prazo**: O fato de a reingestão ter sido concluída com atraso em 2026-05-23 indica que o ritmo do projeto está sofrendo fricção invisível com o tempo de resposta das chaves do cliente. É preciso blindar o cronograma para que o setup da Ingrid (PROJ-002) não seja canibalizado pela cauda longa do Hypercare do Valdece.
2. **O Perigo de Ingestões Manuais Duplicadas**: Como o script `v3_migration.sql` já foi aplicado na base Vanguard e os 61 acórdãos foram populados com os novos campos, qualquer nova alteração manual na tabela antiga gerará inconsistência crônica no momento do dump para a infraestrutura `sa-east-1`. O banco Vanguard deve ser colocado em modo *Read-Only* imediatamente.
3. **Blindagem Sintática contra cp1252 no Windows**: Relembro ao Músculo a regra permanente de encoding do LEDGER. Ao manipular scripts automáticos de dump e migração de dados entre os tenants do Supabase utilizando PowerShell ou Python no ambiente de desenvolvimento, a omissão explícita de `encoding='utf-8'` corromperá os caracteres especiais das ementas jurídicas (como as marcações de parágrafos e o símbolo `≠`), quebrando o threshold semântico da busca.
