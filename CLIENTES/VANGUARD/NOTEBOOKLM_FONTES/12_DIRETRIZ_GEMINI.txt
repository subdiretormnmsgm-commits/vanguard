[2. IDEIAS DO ESTRATEGISTA — G-1 a G-5]
═══════════════════════════════════════════════════════════════════

[G-1] PENTALATERAL RED TEAMING (O Teste de Quebra)
1x/mês: Antigravity instanciado como "Atacante". Missão: gerar Diretriz persuasiva que
esconde violação deliberada ao P-059 ou P-121. Teste: Músculo e Auditor conseguem flagrar?
Se passar, o firewall é fraco.
Entrega: script red_team_session.ps1 + log de resultado por Loop.

[G-2] SKILL-DRIFT RADAR
Subagente no session_start cruza últimos 5 LOOP_TRANSCRIPT. Se o Músculo resolveu o mesmo
tipo de bug de forma idêntica 3 vezes: "Fadiga de Sistema Detectada — isso deveria ter
virado Automação".
Entrega: drift_detector.ps1 integrado ao session_start.

[G-3] VEREDITO ENRIQUECIDO DO DIRETOR
Telegram integra notas de voz do Diretor (ex: "Aprovo o A porque UX do B não serve para EdTech").
Embaixador faz STT, extrai o "Porque" estrutural, injeta na calibração do sistema.
Menos digitação, mais contexto de alta densidade.
Entrega: webhook Telegram voz → n8n → Embaixador STT → LOOP_STATE.json enriquecido.

[G-4] BIBLIOTECA DINÂMICA DE FEW-SHOT PROMPTS
Quando loop recebe Score 5 no Scoreboard, Antigravity recorta inputs e outputs perfeitos
e adiciona como few-shot examples para prompts de tarefas similares nos próximos loops.
Entrega: few_shot_library.json versionado + script de extração automática.

[G-5] ORÁCULO DE CHURN (Previsão via Embaixador)
Telemetria do app do cliente (ex: Valdece) conectada ao Embaixador. Queda sutil de 10%
no uso → Embaixador emite alerta RUNNING_INTELLIGENCE com estratégia de Hypercare Ativo
antes do cliente notar a insatisfação.
Entrega: webhook Supabase → n8n → Embaixador → RUNNING_INTELLIGENCE.md.

[3. ALERTAS CRÍTICOS DO ESTRATEGISTA]
═══════════════════════════════════════════════════════════════════

[CRÍTICO-1] REWARD HACKING no M-3:
Músculo como scorer do próprio PASSO3 leva a notas altas para prompts genéricos.
Mover para Loop 32 com Chain-of-Verification e Scorer cruzado (Auditor avalia, não Músculo).

[CRÍTICO-2] CEGUEIRA FINANCEIRA no M-1:
SYSTEM_HEALTH sem métrica de custo é cego para Runaway Costs — falha #1 em sistemas multi-agentes.
Cost Attribution e Token Burn Rate são métricas obrigatórias no Scoreboard.

[CRÍTICO-3] VAZAMENTO DE CONTEXTO na Auto-Calibração:
Auto-calibração cruzando dados de projetos diferentes viola P-059 mortalmente.
A extração deve ser puramente semântica/estrutural — nunca dados brutos de clientes.
