## PROJETISTA (7º Membro)
**Foco:** Ampliar de projetista estático para motor vivo de Case-Based Reasoning e Adversário Contínuo.

1. **`cbr-rag-orchestrator`**
   - **Origem:** Repositório de skills arquiteturais RAG.
   - **Por que é best-fit:** Destrava a ideia M'P-1. O Projetista precisa recuperar casos por dor, não por data. Esta skill estrutura a busca por vizinhança semântica de problemas, essencial para o histórico virar CBR.
2. **`adversarial-red-teaming`**
   - **Origem:** Frameworks de teste de resiliência.
   - **Por que é best-fit:** Destrava a ideia M'P-2. Transforma o pre-mortem de um checklist estático para um agente que injeta objeções enquanto o projeto está sendo desenhado.
3. **`wbs-eap-generator`**
   - **Origem:** Repositório de gestão de projetos.
   - **Por que é best-fit:** O Projetista deve decompor projetos com rigor. Esta skill estrutura a decomposição hierárquica automática (Work Breakdown Structure).
4. **`notebooklm-studio-director`**
   - **Origem:** Integração de API / Automação Chrome.
   - **Por que é best-fit:** Amplia a capacidade do Projetista de comandar a materialização no NotebookLM com prompts modulares para áudios e infográficos.
5. **`cross-domain-pattern-matcher`**
   - **Origem:** Skills analíticas de correlação.
   - **Por que é best-fit:** Apoia G-1 (Polinização Cruzada de Nichos). Permite que o Projetista identifique que uma solução de EdTech pode ser pivotada para HealthTech.

## EMBAIXADOR DIGITAL (8º Membro)
**Foco:** Ampliar de publicador para autoridade citada por IA (AEO) e sensor de intenção.

1. **`aeo-entity-optimizer`**
   - **Origem:** Frameworks de SEO Generativo / AEO.
   - **Por que é best-fit:** Destrava M'D-1. Estrutura posts e artigos do LinkedIn em blocos semânticos densos, otimizados para serem ingeridos e citados por Perplexity e Google AI Overviews.
2. **`signal-based-lead-scorer`**
   - **Origem:** Vendas B2B Account-Based.
   - **Por que é best-fit:** Destrava M'D-2. Avalia engajamentos não pelo volume, mas cruzando o perfil do lead com o ICP e a urgência regulatória atual.
3. **`linkedin-b2b-authority-builder`**
   - **Origem:** Repositório de conteúdo B2B de alto ticket.
   - **Por que é best-fit:** Garante o alinhamento com os pilares de conteúdo (Thought Leadership, Business Case, Prova Técnica) sem viés publicitário.
4. **`objection-to-content-converter`**
   - **Origem:** Skills de retroalimentação de mercado.
   - **Por que é best-fit:** Apoia o cruzamento com o Projetista. Transforma as objeções detectadas no mercado em insumo direto para novos conteúdos de mitigação de risco.
5. **`icp-behavioral-auditor`**
   - **Origem:** Skills de análise de perfil social.
   - **Por que é best-fit:** Automatiza a Fase 4 do seu método (Validação de ICP), medindo o CTR real contra a vaidade dos likes baseando-se no porte da empresa do engajador.

## DETECTOR DE DERIVA (Coadjuvante)
**Foco:** Ampliar de leitor reativo para guardião preditivo (Policy as Code e Baseline Semântico).

1. **`policy-as-code-validator`**
   - **Origem:** Integração CI/CD e Compliance.
   - **Por que é best-fit:** Destrava M'X-A. Permite traduzir os princípios mais objetivos do LEDGER em regras executáveis para hooks de pre-commit.
2. **`baseline-promise-tracker`**
   - **Origem:** Monitoramento de SLA / Semantic Drift.
   - **Por que é best-fit:** Destrava M'X-B. Compara a versão atual de um artefato com sua "promessa" inicial, identificando perda de foco ou inflação de escopo.
3. **`obsidian-graph-query`**
   - **Origem:** Kepano Obsidian Skills.
   - **Por que é best-fit:** Expande a capacidade de uso de backlinks e tags do Obsidian, detectando derivas estruturais que grep não encontra.
4. **`ledger-consistency-checker`**
   - **Origem:** Skills de auditoria de base de conhecimento.
   - **Por que é best-fit:** Verifica contradições cruzadas entre princípios recém-inseridos e princípios antigos, garantindo que a própria régua não sofra deriva.
5. **`pre-commit-compliance-hook`**
   - **Origem:** Ferramentas de Git hooks.
   - **Por que é best-fit:** Atua junto com o policy as code para travar commits que descaradamente quebrem R-01 ou P-059, antes mesmo de entrar no repositório.
