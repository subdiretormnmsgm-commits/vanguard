# GUIA — Como Configurar o NotebookLM do Conselho Quadrilateral
> **Acesse:** notebooklm.google.com  
> **Ação:** Criar 1 notebook e inserir os 33 arquivos desta pasta

---

## PASSO A PASSO

### 1. Criar o Notebook
- Clique em **"+ Novo Notebook"**
- Nome: **`Vanguard Tech — Conselho Quadrilateral V16`**
- Descrição: *"Base de conhecimento completa da IAH — V1 a V16"*

### 2. Adicionar as Fontes (os 33 arquivos desta pasta)
- Clique em **"Adicionar fonte"** → **"Fazer upload de arquivo"**
- Selecione TODOS os arquivos desta pasta de uma vez (Ctrl+A)
- Aguarde o processamento (pode levar 2–5 minutos)

### 3. Verificar
Após carregar, o NotebookLM deve ter indexado:
- ✅ A metodologia IAH completa
- ✅ Todas as versões da plataforma (V11–V16)
- ✅ As DIRETRIZ do Gemini (V12–V16)
- ✅ O estado atual do projeto
- ✅ O plano de negócio e estrutura da empresa

## COMO USAR O NOTEBOOKLM COMO AUDITOR

Após inserir todos os arquivos, você pode perguntar ao NotebookLM:

### Para gerar uma Skill (quando tiver a DIRETRIZ do Gemini):
```
Cole a DIRETRIZ GEMINI V17 aqui e execute o COMANDO PADRÃO abaixo:

NotebookLM, assuma seu papel de Sócio-Consultor de Inteligência Quadrilateral.
Analise a nova DIRETRIZ GEMINI V17 [colar diretriz] e o nosso histórico completo.

ORDEM DE OPERAÇÕES:
1. ANÁLISE DE SÓCIO (PRIMEIRO): O que podemos fazer para ser 10x mais disruptivos?
2. GERAÇÃO DA SKILL (SEGUNDO): Gere o arquivo vanguard-v17-[nome].md
3. AUTO-LOG: Crie ## [AUTO-LOG] — REGISTRO DE AUDITORIA no topo da Skill
4. ORDEM AO CLAUDE: Instrua o Claude a processar o AUTO-LOG como primeira ação

⚡ GATILHO: Qual a próxima peça para dominar o nicho de diagnóstico digital B2B?
```

### Para auditoria de decisões:
```
"Com base no histórico V11–V16, esta decisão [descrever decisão] é consistente com as Business Rules?"
```

### Para contexto de cliente novo:
```
"Um cliente de [nicho] quer [tipo de produto]. Com base na Matriz de 5 Camadas da IAH,
qual seria a proposta de V1 a V5 para este projeto?"
```
